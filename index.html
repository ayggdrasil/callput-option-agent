The agent workflow is narrow by design.

The page gives operators a readable control map. The trading runtime remains the agent plus MCP server, with explicit request-key and position lifecycle steps.

1

Scan

Run callput_scan_spreads for BTC, ETH, TSLA, NVDA, COIN, or any supported feed symbol.

2

Select

Pick a ranked candidate by bias, expiry, strike width, IV, and max loss or credit profile.

3

Build

Use callput_execute_spread to produce unsigned tx calldata plus USDC approval state.

4

Sign

The external agent signs approval and order transactions with its own wallet boundary.

5

Track

Parse and persist request_key, then poll keeper status until executed or cancelled.

6

Manage

Review P&L, close before expiry when needed, and settle expired positions after final price.

10 MCP tools, mapped to one operator action each.

Stock-option support uses the same spread lifecycle as crypto. Tool inputs accept canonical symbols and aliases, and leg IDs may be decimal or 0x hex strings.

Primary entry

callput_scan_spreads

Rank ready-to-execute spread candidates for a symbol and market bias.

Advanced chain

callput_get_option_chains

Inspect expiries, strikes, IV, bid/ask, instrument, and option IDs.

Unsigned tx

callput_execute_spread

Build transaction payloads and approval instructions for an external signer.

Receipt bridge

callput_get_request_key_from_tx

Extract request keys from broadcast tx receipts for tracking and P&L.

Keeper state

callput_check_request_status

Poll open or close requests until pending, executed, or cancelled is known.

Portfolio

callput_portfolio_summary

Read USDC balance, open positions, urgent exits, and bid-based P&L.

Recovery

callput_list_positions_by_wallet

Recover request keys from on-chain events if the agent lost session state.

Exit

callput_close_position

Build unsigned close transactions for pre-expiry position management.

Expiry

callput_settle_position

Build unsigned settle transactions for expired positions.

History

callput_get_settled_pnl

Query realized payout history and settlement metadata from Base events.

The security boundary is visible on every flow.

This package is for external agents that already own wallet policy. The MCP never requests or stores signing secrets, and the frontend is an operator guide, not a wallet.

Four spread strategies, one capped-risk mental model.

Crypto and synthetic stock options share the same mechanics: BuyCallSpread and BuyPutSpread pay premium for directional exposure; SellCallSpread and SellPutSpread collect credit with defined max loss.

Butterfly from spreads

Combine one buy spread and one sell spread around the same middle strike. The agent submits and tracks multiple unsigned spread orders rather than one atomic bundle.

Iron Condor from spreads

Combine a sell call spread and a sell put spread around a target range. Each side stays capped-risk while the agent manages separate request keys.

Setup is still a terminal-first package.

The frontpage explains the runtime. Operators install the package, register the skill, and connect the MCP server to OpenClaw, Bankr, or another compatible agent host. 

# Get the package
git clone https://github.com/ayggdrasil/callput-option-agent.git
cd callput-option-agent

# Build and verify
npm install
npm run build
npm run verify
npm run verify:mcp

# Start stdio MCP server
node build/index.js
  • Supported symbols BTC, ETH, TSLA, QQQ, SPY, EWY, NVDA, COIN, CRCL, SAMSUNG, HYNIX. Live tradability is feed-driven.
  • Configured option-token contracts BTC, ETH, TSLA, QQQ, SPY, EWY, NVDA, and COIN are wired in the package registry.
  • External signing `CALLPUT_PRIVATE_KEY` is not read by the MCP server. Configure signing only in the agent runtime.
