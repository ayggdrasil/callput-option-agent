## Design Context

### Users
External agent operators and developers connecting Callput Lite MCP to OpenClaw, Bankr, and compatible runtimes. They are technically experienced, but may be new to this package and need a short path from repository link to first unsigned transaction payload.

### Brand Personality
Institutional, precise, and operational. The interface should feel like a trustworthy runbook for high-stakes agent trading, not a decorative trading dashboard.

### Aesthetic Direction
Use an operator-runbook layout with calm dark neutrals, restrained accents, dense but readable command blocks, and explicit safety boundaries. Avoid generic neon dashboard styling, hero metric clutter, and decorative glassmorphism.

### Design Principles
- Put the activation path first: clone, register MCP, run the first prompt.
- Keep signing and private-key boundaries visible near every trading action.
- Prefer copyable, executable snippets over explanatory prose.
- Use market and strategy visuals as supporting evidence, not the primary story.
- Design for experienced operators who want value in five minutes without hiding safety details.
